# Methodology

## Overview
This project simulates gravitational-wave (GW) and gamma-ray burst (GRB) trigger populations, then applies simple temporal and spatial coincidence tests to identify joint candidates.

## Simulation
- GW triggers are simulated in `src/simulate_gw.py`.
- GRB triggers are simulated in `src/simulate_grb.py`.
- Each trigger includes:
  - `time_gps`: trigger time in GPS seconds
  - `ra`, `dec`: sky location in radians
  - `snr` (GW) or `fluence` (GRB)
- RA is drawn uniformly in `[0, 2*pi]` and Dec in `[-pi/2, pi/2]`.

## Coincidence Test
- Coincidences are evaluated in `src/coincidence.py` using:
  - Time window: `DELTA_T` seconds
  - Sky window: `DELTA_OMEGA` radians
- Angular separation is computed via spherical cosine law.
- A pair is coincident if both time and angular thresholds are satisfied.

## Ranking Statistic
- A simple ranking statistic is defined in `src/statistics.py` as `p_signal / p_noise`.
- This is a placeholder for more realistic detection statistics.

## Outputs
- Simulated triggers: `data/simulated/gw_triggers.csv`, `data/simulated/grb_triggers.csv`
- Coincidence outputs: `data/results/coincident_events.csv`
- Summary statistics: `data/results/summary_stats.txt`
- Plots: `figures/` (generated by `src/plotting.py`)

## Notes and Limitations
- Distributions are simplified and not astrophysically calibrated.
- The pipeline is meant for conceptual understanding and testing workflows.
