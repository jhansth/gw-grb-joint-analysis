# GW-GRB Joint Analysis - Project Status

## Status
Project is clean and ready for development in WSL2 with a single virtual environment.

## Current Structure (clean)
```
gw-grb-joint-analysis/
|-- src/                       # Python source code
|-- notebooks/                 # Jupyter notebooks
|-- data/
|   |-- simulated/             # Simulated triggers (CSV)
|   `-- results/               # Coincidence outputs + summary stats
|-- figures/                   # Saved plots
|-- docs/                      # Documentation
|-- run_pipeline.sh            # One-command pipeline
|-- setup_wsl.sh               # WSL2 setup script
|-- requirements.txt           # Dependencies
|-- README.md                  # Main documentation
|-- QUICKSTART.md              # Quick usage guide
`-- WSL2_SETUP.md              # WSL2-specific setup
```

## Environment
- WSL2 is required (healpy does not build on Windows)
- Virtual environment: `~/venv_linux`
- Activate with: `source ~/venv_linux/bin/activate`

## How to Run
- Full pipeline: `bash run_pipeline.sh`
- Individual scripts: `python src/simulate_gw.py`, `python src/simulate_grb.py`, `python src/coincidence.py`, `python src/statistics.py`, `python src/plotting.py`
- Notebook: `jupyter notebook notebooks/exploratory_analysis.ipynb`

## Outputs
- `data/simulated/`: GW/GRB trigger CSVs
- `data/results/`: coincidence outputs and summary stats
- `figures/`: saved plots from healpy

## Notes
- Generated artifacts are tracked as sample outputs and can be regenerated by running the pipeline.
- If you use a local venv inside the repo, it is ignored by `.gitignore`.
